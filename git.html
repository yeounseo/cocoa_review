What is Git ?

  1. 깃은 우리가 파일에서 만든 변경사항을 추적하는 시스템이라고 할 수 있다.
  2. 워드, 이미지, 포토샵, 혹은 소스코드!
  3. html , css , Javascript 등등

깃의 특징

  • 변경사항들을 다 알려주고, 백업인것인지 언제 무엇을 바꾸었는지 알 수 있다.
  • 우리가 push 해서 올리면, 모든 변경사항과 내용들이 클라우드에 서버에 저장된다.

깃 Verson Control

Version Control: Repositories, Commits, Branches

  1. repository : 소스코드를 저장하는 폴더라고 이해하자
  2. commit : 파일 변경을 기록함!
  3. branches : 나무의 가지(branch)

처음 디폴트로 생성되는 것은 master이다. master branch에는 모든 commit이 반영 되어있다.

마스터 브랜치를 건드리고 싶지 않다면?

새로운 브랜치를 마치 가지처럼 만든다.

실험을 해보고, 마스터 브랜치는 건드리지 않고, 새로운 기능도 추가 해보려고 할때 적합하다.

유저들은 마스터를 이용 , 그 외 수정 사항이나 추가사항은 브랜치로 작업하여, 추후 통합

대표적인 브랜치 이용

  1. 브랜치를 생성
  2. 마스터는 그대로 내버려둔다
  3. 브랜치에서 새로운 작업 완료
  4. 브랜치를 마스터에 결합
요약
  1. repository는 코드를 저장하는 폴더, 깃은 그것을 추적
  2. commit은 깃 파일(변경사항)의 보관함이여서 무엇이 변경되었는지를 기록
  3. branch는 마스터의 복사판으로, 새로운 것을 테스트해보고 싶을 때 사용하면 된다.
